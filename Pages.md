# 아이디어 공모 페이지
0. 아이디어 공모 페이지에 대한 설명을 이미지로 올릴 예정이다. LayOut을 화면의 절반 정도로 맞춰줘라.
1. 로그인이 되어 있지 않다면 로그인 페이지로 이동한다. 로그인 이후 다시 아이디어 공모 페이지로 리다이렉트 한다. 로그인 페이지에는 기존에 개발되어있는 카카오 Oauth 로그인, 구글 Oauth 로그인, 이메일 로그인 페이지로 이동 가능한 버튼이 있다. (로그인 페이지는 별도의 페이지를 따로 만들어라) redirect 관련 변수에 아이디어 공모 페이지 uri를 추가한다.
2. 이미지 업로드가 가능한 Multipart-Form을 만든다.
3. 1번을 입력 받아 Backend API에 요청 보낼 수 있게한다.
4. CSS는 BackgroundColor는 White기반, Point-Color는 Light Blue 기반으로 한다.
5. 아이디어 공모 관련 Entity는 기본 식별자 id, User, memoImage(Multiparfile로 받아서 S3에 저장, 별도 Entity로 만들어서 메타데이터 저장할 것임.), createdAt 필요, 추가적으로 필요한 내용이 있다고 판단된다면 필드 추가해주기 바람.
6. MemoImage 저장 메타데이터 목록 : id, originalFilename, s3Key, fileSize
7. 아이디어 공모 신청 후에는 `아이디어 공모 신청 감사 페이지`로 이동한다.

# 전역 설정(Global Configuration)
0. Vue 관련 코드를 수정하게 되면. 전역 설정(Global Configuration)을 항상 참조하여 위배되지 않도록 한다.
1. 리다이렉트 관련 변수는 `zdoPostLoginRedirectPath`으로 전역적으로 통일한다. 이 변수는 Local Storage에 저장된다.
2. 리다이렉트(Redirect) 동작이 실행된 후 `zdoPostLoginRedirectPath` 변수는 null 처리 된다.
3. 브라우저가 종료되는 상황(강제종료, 일반적인 종료 등 모든 종료 상황을 포함한다.)이 발생하면 Local Storage를 Empty 상태로 만든다. 모든 변수에 대해 null 처리 한다.
4. 기본 CSS는 항상 BackgroundColor는 White Point-Color는 Light Blue이다.
5. Vue로 작성한 모든 페이지에 대해 i18n을 항상 적용한다.(한국어,영어,중국어, 일본어) Drop-Down-Select-Box 형태로 만든다. (공간 효율성을 위해.)
6. 응답 구조의 Message는 간결하게 보내라.
